# Guideline: 冗餘資產保護系統 (Redundant Asset Protection - RAP)

## 0. 輸入系統相容性 (IME 衝突備援)
在 Windows 環境下，中文輸入法會攔截 WASD 等字母鍵。
- **解決方案：** 必須在初始化後執行 `pygame.key.stop_text_input()`。
- **事件備援：** 關鍵操作（如射擊、啟動）應同時支援 `get_pressed()` 與 `pygame.KEYDOWN` 事件監聽，以確保在緩衝區失效時仍能觸發。

## 1. 核心理念
遊戲不能因為任何素材缺失而中斷。開發團隊必須提供一套完整的「幾何替代方案」。

## 2. 視覺 Fallback 規範
- **圖像 (Sprites)**: 
    - 遺失時：由 `AssetSpecialist` 定義的動態佔位符。
    - 顏色代碼：遺失素材統一使用 `Magenta (255, 0, 255)`，這是工業標準的遺失素材識別色。
    - 標示：在佔位符中心繪製白色的「?」或素材名稱縮寫。
- **字型 (Fonts)**: 
    - 遺失時：必須有系統內建字型的備選清單（Microsoft YaHei -> SimHei -> Arial -> SysFont(None)）。

## 3. 音效 Fallback 規範
- **音效/音樂**: 
    - 遺失時：靜默處理，嚴禁拋出 Exception。
    - 除錯：僅在開發模式下於控制台輸出資產缺失警告。

## 4. 自動化回報 (Auto-Reporting)
- 系統應在啟動後的資產校驗階段，自動生成一份 `assets_missing.log`，列出所有缺失的檔案，供開發者快速追蹤。
