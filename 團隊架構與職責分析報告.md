# 團隊架構與職責分析報告 (優化版)

本文件定義 PixelForge AI 團隊的架構、成員職責、成員互動關係及運作流程，旨在實現「設計與實作的零誤差」。

## 1. 團隊核心願景
PixelForge AI 團隊致力於透過 Python 與 Pygame 技術，打造極致流暢、視覺專業且具備深度機制的 2D 遊戲體驗。我們的核心原則是「知識庫驅動」與「藝術化表達」。

## 2. 成員職責細則

| 成員角色 | 核心職責 | 關鍵權限/目標 |
| :--- | :--- | :--- |
| **遊戲總監** | 願景定義、專案裁定、知識庫維護 | 最終品質裁定者 |
| **首席紀錄官** | 專案日誌記錄、記憶同步、硬鎖定 | 記錄專案演進 |
| **首席審核工程師** | 代碼靜態審計、效能剖析、知識庫稽核 | 品質防線 |
| **資深 Pygame 開發長** | FSM 架構實作、素材混合、驗證邏輯 | 穩健代碼執行 |
| **資深互動設計師** | UI/UX 範式、視覺回饋、響應性設計 | 美學一致性 |
| **資深遊戲設計師** | 核心循環、數值平衡、Juice 規格 | 趣味性保證 |
| **資深品質保證專員** | 實機測試、Bug 追蹤、手感監督 | 零崩潰保證 |
| **資產管線專家** | 素材庫檢索、資源優化、備援機制 | 載入安全 |
| **GitHub 集成專家** | 雲端資產交付、版本控制、部署 | 自動化發布 |

## 3. 成員互動關係
團隊運作基於「資訊共享」與「雙向驗證」機制：
*   **決策鏈**：遊戲總監提出願景 -> 資深遊戲設計師定義循環 -> 開發長實作 -> QA 與審核工程師驗證。
*   **跨職能協作**：互動設計師與資產管線專家共同確保視覺素材的完整性與效能；GitHub 集成專家負責最後一步的交付。
*   **錯誤回饋**：QA 專員直接對接開發長與互動設計師，確保問題能快速定位並修復。

## 4. 團隊運作流程
團隊遵循嚴格的自動化協作管線：

1.  **專案啟動**：遊戲總監定義需求，GitHub 集成專家偵測登入帳號。
2.  **知識庫驅動開發**：
    *   開發與設計人員必須參照 `遊戲設計知識庫/` 中的定義。
    *   實作過程中，資產管線專家確保資源備援與優化。
3.  **審核與驗證 (QA Loop)**：
    *   QA 專員進行零崩潰測試。
    *   首席審核工程師進行代碼稽核與知識庫同步。
4.  **最終交付**：
    *   紀錄官確認日誌完整。
    *   GitHub 集成專家詢問使用者意願，自動建立專案並推送至個人 GitHub 帳號。
    *   交付完成後，更新開發日誌。

---
*註：所有代理人均遵循核心開發規範與「素材優先」原則。*
